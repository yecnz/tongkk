# 섹션별 핵심 정리

## 1. 알고리즘이란 (Chapter 01)

### [알고리즘의 정의]
- 어떤 작업을 수행하기 위한 과정을 애매하지 않게 기술한 것
- 어떤 작업을 수행하기 위해 입력을 받아 출력을 만들어내는 과정을 애매하지 않게 기술한 것

### [바람직한 알고리즘의 특징]
- **명확성 (Clarity)**
    - 이해하기 쉽고 가능하면 간명해야 함
    - 지나친 기호적 표현은 명확성을 떨어뜨릴 수 있음 (명확성을 해치지 않는 범위 내에서 일반언어 사용 가능)
- **효율성 (Efficiency)**
    - 동일한 문제를 해결하더라도 알고리즘에 따라 수행 시간이 수백만 배 이상 차이 날 수 있음

### [알고리즘의 예시: 100명의 학생 시험점수 최댓값 찾기]
- **입력**: 100개의 점수
- **출력**: 입력된 100개의 점수들 중 최댓값
- **알고리즘**: (해당 과정 기술)

### [알고리즘의 학습 가치]
- **생각하는 방법의 훈련**
    - 문제 자체를 해결하는 알고리즘을 배우는 것보다, 그 과정에 담긴 ‘생각하는 방법’을 배우는 것이 더 중요함
    - 미래의 다른 문제를 해결하기 위한 '생각의 빌딩블록(Building Block)'을 제공함
- **자료구조의 확장**
    - 선행 과목: 프로그래밍, 자료구조
    - 관계: 자료구조는 건축의 건축 자재나 모듈, 자동차 제작의 부품이나 모듈과 같은 역할을 하며, 알고리즘은 이를 확장함

## 2. 알고리즘 표기법

### [알고리즘 표기법의 종류 및 특징]
- **자연어를 이용한 서술적 표현**
    - 특징: 서술적임
    - 단점: 쓰는 사람에 따라 일관성이나 명확성을 유지하기 어려우며, 누구나 쉽게 이해하고 쓸 수 있는 표현에는 한계가 있음
- **순서도를 이용한 도식화**
    - 특징: 명령의 흐름을 쉽게 파악할 수 있음
    - 단점: 복잡한 알고리즘을 표현하는 데는 한계가 있음
- **프로그래밍 언어를 이용한 구체화**
    - 특징: 추가적인 구체화 과정이 필요 없음
    - 단점: 해당 언어를 모르면 이해하기 어려움, 다른 언어로 개발 시 변환이 필요하므로 범용성이 떨어짐
- **가상코드 [Pseudo-Code]를 이용한 추상화**
    - 특징: 직접 실행은 불가능하지만, 일반적인 프로그래밍 언어와 형태가 유사하여 프로그래밍 언어로 구체화하기 쉬움

### [이 책에서 사용하는 알고리즘 표기법 규칙]
- 함수 시작 지점 명시
- 문장 뒤 세미콜론(;) 생략
- `If` 문 뒤의 `then` 생략
- `for` 루프에 속하는 문장들은 들여쓰기 적용
- 주석 사용

# 열거형 암기 포인트
- **바람직한 알고리즘의 조건 2가지**: 명확성, 효율성
- **알고리즘 표기법의 종류 4가지**: 자연어를 이용한 서술적 표현, 순서도를 이용한 도식화, 프로그래밍 언어를 이용한 구체화, 가상코드[Pseudo-Code]를 이용한 추상화
- **본 교재의 표기법 규칙 5가지**: 함수 시작, 세미콜론 생략, If 뒤 then 생략, for 루프 문장 들여쓰기, 주석

# 비교 포인트
| 구분 | 자연어 표현 | 순서도 | 프로그래밍 언어 | 가상코드(Pseudo-Code) |
| :--- | :--- | :--- | :--- | :--- |
| **장점** | 누구나 이해하기 쉬움 | 명령 흐름 파악 용이 | 추가 구체화 불필요 | 프로그래밍 언어로 변환 용이 |
| **단점** | 일관성/명확성 유지 어려움 | 복잡한 알고리즘 표현 한계 | 언어 의존성 높음(범용성 낮음) | 직접 실행 불가능 |

# 시험 직전 체크리스트
- [ ] **알고리즘**: 작업을 수행하기 위해 입력을 받아 출력을 만들어내는 과정을 애매하지 않게 기술한 것
- [ ] **명확성**: 알고리즘이 이해하기 쉽고 간명하며, 지나친 기호 사용을 지양해야 하는 특성
- [ ] **효율성**: 알고리즘의 성능(수행 시간 등)이 극명하게 차이 날 수 있음을 나타내는 특성
- [ ] **빌딩블록(Building Block)**: 알고리즘 학습을 통해 얻게 되는, 미래 문제 해결을 위한 사고의 기초 단위
- [ ] **자료구조**: 알고리즘의 바탕이 되는 건축 자재나 부품과 같은 개념
- [ ] **가상코드(Pseudo-Code)**: 프로그래밍 언어로 구현하기 쉽도록 추상화된 표기법
- [ ] **범용성**: 프로그래밍 언어 표기법이 가질 수 있는 한계점(다른 언어 사용 시 변환 필요)
- [ ] **들여쓰기**: `for` 루프 내부 문장을 구분하기 위한 표기 규칙
- [ ] **세미콜론 생략**: 본 교재에서 채택한 특정 표기 규칙 중 하나

# 한 줄 요약
알고리즘은 입력을 출력으로 변환하는 명확하고 효율적인 작업 과정을 기술한 것으로, 논리적 사고의 틀(Building Block)을 제공하며 다양한 표기법(자연어, 순서도, 프로그래밍 언어, 가상코드)을 통해 표현될 수 있다.
