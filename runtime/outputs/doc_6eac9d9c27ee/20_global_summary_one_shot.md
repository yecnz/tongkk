# 섹션별 핵심 정리

## 1. 순환 (Recursion)
- **핵심 정의**
    - 알고리즘이나 함수가 수행 도중에 자기 자신을 다시 호출하여 문제를 해결하는 기법
- **핵심 설명**
    - 정의 자체가 순환적으로 되어 있는 경우에 적합한 방법

## 2. 팩토리얼 프로그래밍 (Factorial Programming)
- **핵심 정의**
    - $n! = \begin{cases} 1 & n = 0 \\ n \times (n-1) \times \dots \times 1 & n \ge 1 \end{cases}$
- **순환 알고리즘의 구조**
    - **순환을 멈추는 부분 (Base Case):** `if (n <= 1) return 1;`
    - **순환 호출을 하는 부분 (Recursive Step):** `else return n * factorial(n-1);`
- **특징**
    - 만약 순환을 멈추는 부분이 없다면: 시스템 오류가 발생할 때까지 무한정 호출하게 됨
- **비교: 순환 vs 반복**
    - 대부분의 순환은 반복문(Iteration)으로 바꾸어 작성할 수 있음
    - 팩토리얼의 반복적 구현: `for (k=n; k>0; k--) v = v*k;`

## 3. 거듭제곱 값 프로그래밍 (Power Function Programming)
- **핵심 정의**
    - 숫자 $x$의 $n$제곱 값을 구하는 문제: $x^n$
- **순환적인 알고리즘 (Recursive Algorithm)**
    - `if n == 0 then return 1;`
    - `else if n이 짝수 then return power(x*x, n/2);`
    - `else if n이 홀수 then return x * power(x*x, (n-1)/2);`
- **비교: 반복적인 방법 vs 순환적인 방법**
    - **시간 복잡도:**
        - 반복적인 함수 (`slow_power`): $O(n)$
        - 순환적인 함수 (`power`): $O(\log n)$
    - **실제 수행 속도 (예시):**
        - 반복적인 방법: 7.17초
        - 순환적인 방법: 0.47초

## 4. 피보나치 수열의 계산 (Fibonacci Sequence Calculation)
- **핵심 정의**
    - $fib(0) = 0, fib(1) = 1$
    - $fib(n) = fib(n-1) + fib(n-2)$ (단, $n \ge 2$)
- **순환 호출 사용 시의 비효율성 (특징)**
    - 같은 항이 중복해서 계산됨
    - 예: $fib(6)$을 호출하면 $fib(3)$이 4번이나 중불해서 계산됨
    - $n$이 커질수록 이러한 현상은 더 심해짐
- **반복적 구현 (특징)**
    - `pp`(이전 이전 항), `p`(이전 항), `result`(현재 항) 변수를 사용하여 반복문으로 계산

## 5. 하노이 탑 문제 (Tower of Hanoi Problem)
- **핵심 정의**
    - 막대 A에 쌓여있는 원판 $n$개를 막대 C로 옮기는 문제
- **문제의 규칙 (4가지)**
    1. 한 번에 하나의 원판만이 이동할 수 있다.
    2. 맨 위에 있는 원판만이 이동할 수 있다.
    3. 크기가 작은 원판 위에 큰 원판이 쌓일 수 없다.
    4. 중간의 막대(B)를 임시적으로 이용할 수 있으나 앞의 조건들을 지켜야 한다.
- **일반적인 해결 전략 (3단계)**
    1. $n-1$개의 원판을 A에서 B로 옮긴다 (C를 임시 버퍼로 사용).
    2. A에 있는 가장 큰 원판을 C로 옮긴다.
    3. B에 쌓여있는 $n-1$개의 원판을 C로 옮긴다 (A를 임시 버퍼로 사용).

# 열거형 암기 포인트
- **하노이 탑 문제의 규칙 4가지:** 1) 한 번에 하나의 원판 이동, 2) 맨 위 원판만 이동 가능, 3) 큰 원판이 작은 원판 위에 쌓일 수 없음, 4) 중간 막대 이용 시 규칙 준수 필수.
- **하노이 탑의 일반적인 해결 단계:** 1) $n-1$개를 A $\to$ B로 이동, 2) 가장 큰 원판 A $\to$ C로 이동, 3) $n-1$개를 B $\to$ C로 이동.

# 비교 포인트
- **거듭제곱 계산 방식 비교**
    - **반복적 방법 (`slow_power`):** 시간 복잡도 $O(n)$, 상대적으로 느림.
    - **순환적 방법 (`power`):** 시간 복잡도 $O(\log n)$, 상대적으로 매우 빠름.
