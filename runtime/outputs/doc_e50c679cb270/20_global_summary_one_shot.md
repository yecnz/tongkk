# 전체 흐름
- 순환(Recursion)의 정의 및 적합한 상황
- 팩토리얼(Factorial)을 통한 순환의 기본 구조와 구현
- 순환과 반복(Iteration)의 관계 및 팩토리얼의 반복적 구현
- 거듭제곱(Power) 계산을 통한 순환의 효율성 분석 (시간 복잡도 비교)
- 피보나치 수열(Fibonacci Sequence)을 통한 순환의 비효율성 사례 분석
- 하노이 탑(Tower of Hanoi) 문제를 통한 순환 알고리즘의 적용 및 규칙

# 섹션별 핵심 정리

## 1. 순환 (Recursion)
- **핵심 정의**
    - 알고리즘이나 함수가 수행 도중에 자기 자신을 다시 호출하여 문제를 해결하는 기․기법
- **특징**
    - 정의 자체가 순환적으로 되어 있는 경우에 적합한 방법

## 2. 팩토리얼 프로그래밍 (Factorial Programming)
- **핵심 정의 (팩토리얼의 정의)**
    - $n! = n \times (n-1) \times \dots \times 1$ ($n \ge 1$ 일 때)
    - $n! = 1$ ($n = 0$ 일 때)
- **순환 알고리즘의 구조**
    - 순환을 멈추는 부분 (Base Case): `if (n <= 1) return 1;`
    - 순환 호출을 하는 부분 (Recursive Step): `else return n * factorial(n-1);`
- **주의 사항**
    - 만약 순환 호출을 멈추는 부분이 없다면, 시스템 오류가 발생할 때까지 무한정 호출하게 됨

## 3. 거듭제곱 값 프로그래밍 (Power Function)
- **핵심 설명**
    - 숫자 $x$의 $n$제곱 값을 구하는 문제 ($x^n$)
    - 순환적인 방법이 더 효율적인 예제
- **순환적인 알고리즘의 로직**
    - `if n == 0`: `return 1`
    - `else if n이 짝수`: `return power(x*x, n/2)`
    - `else if n이 홀수`: `return x * power(x*x, (n-1)/2)`
- **비교 대상 (시간 복잡도 및 성능)**
    | 비교 항목 | 반복적인 함수 (`slow_power`) | 순환적인 함수 (`power`) |
    | :--- | :--- | :--- |
    | **시간 복잡도** | $O(n)$ | $O(\log n)$ |
    | **실제 수행 속도** | 7.17초 | 0.47초 |

## 4. 피보나치 수열의 계산 (Fibonacci Sequence)
- **핵심 정의**
    - $fib(0) = 0$
    - $fib(1) = 1$
    - $fib(n) = fib(n-1) + fib(n-2)$ (그 외의 경우)
- **순환 호출 사용 시의 비효율성 (특징)**
    - 같은 항이 중복해서 계산됨 (예: $fib(6)$ 호출 시 $fib(3)$이 4번 중복 계산됨)
    - $n$이 커질수록 이러한 현상은 더 심해짐

## 5. 하노이 탑 문제 (Tower of Hanoi)
- **핵심 정의**
    - 막대 A에 쌓여 있는 원판 $n$개를 막대 C로 옮기는 문제
- **문제의 규칙 (4가지 조건)**
    1. 한 번에 하나의 원판만 이동할 수 있다.
    2. 맨 위에 있는 원판만 이동할 수 있다.
    3. 크기가 작은 원판 위에 큰 원판이 쌓일 수 없다.
    4. 중간의 막대(B)를 임시적으로 이용할 수 있으나 앞의 조건들을 지켜야 한다.
- **일반적인 해결 방식 (알고리즘)**
    1. $n-1$개의 원판을 A에서 B로 옮긴다 (C를 임시 버퍼로 사용).
    2. A에 있는 가장 큰 원판을 C로 옮긴다.
    3. B에 있는 $n-1$개의 원판을 C로 옮긴다 (A를 임시 버퍼로 사용).

# 열거형 암기 포인트
- **순환 알고리즘의 구조 2가지**: 순환을 멈추는 부분, 순환 호출을 하는 부분
- **하노이 탑 문제의 규칙 4가지**: 한 번에 하나의 원판 이동, 맨 위 원판만 이동 가능, 큰 원판이 작은 원판 위로 적재 불가, 중간 막대 임시 이용 가능(조건 준수 시)

# 비교 포인트
- **거듭제곱 계산 방식 비교**
    - **반복적 방법 (`slow_power`)**: 시간 복잡도 $O(n)$, 상대적으로 느린 속도
    - **순환적 방법 (`power`)**: 시간 복잡도 $O(\log n)$, 상대적으로 빠른 속도
- **순환 vs 반복 (일반론)**
    - 대부분의 순환은 반복으로 바꾸어 작성할 수 있음

# 시험 직전 체크리스트
- [ ] **순환(Recursion)**: 자기 자신을 다시 호출하여 문제를 해결하는 기법
- [ ] **Base Case (순환 멈춤 조건)**: 순환을 종료하기 위해 반드시 필요한 조건 (예: 팩토리얼의 $n \le 1$)
- [ ] **무한 순환**: 멈춤 조건이 없을 때 시스템 오류가 발생할 때까지 계속되는 상태
- [ ] **시간 복잡도 $O(\log n)$**: 거듭제곱 계산 시 순환적 방법이 갖는 효율적인 복잡도
- [ ] **시간 복잡도 $O(n)$**: 거듭제곱 계산 시 반복적 방법이 갖는 복잡도
- [ ] **피보나치 수열의 중복 계산**: 순환 호출 시 동일한 항이 여러 번 계산되어 효율성이 떨어지는 현상
- [ ] **하노이 탑의 핵심 제약**: 큰 원판이 작은 원판 위에 올 수 없음
- [ ] **하노이 탑의 재귀적 로직**: $n-1$개 이동 $\to$ 큰 원판 이동 $\to$ $n-1$개 이동 순서
